Full end-to-end flow: Contiki Node -> MQTT Broker -> Backend -> WebSocket -> Frontend UI

Stage 1 / 5

Stage 1: Contiki Native Node (Data Producer)

Contiki native process runs in Docker with TUN interface and publishes telemetry every 2 seconds to iot-2/evt/status/fmt/json. 

# run-contiki.sh (startup path)
CCACHE_DISABLE=1 make -C "${EXAMPLE_DIR}" TARGET=native clean all
exec "${EXAMPLE_DIR}/build/native/mqtt-client.native" --prefix "${TUN_PREFIX}" -t "${TUN_DEV}"

// mqtt-client.c (publish intent)
mqtt_publish(&conn, NULL, pub_topic, (uint8_t *)app_buffer,
             strlen(app_buffer), MQTT_QOS_LEVEL_0, MQTT_RETAIN_OFF);

Stage 2: Mosquitto Broker (Message Router)

Broker receives publish packets from Contiki client and routes them to subscribers matching the topic. Publisher and subscriber are decoupled. 

# compose
mosquitto:
  image: eclipse-mosquitto:2
  ports:
    - "1883:1883"

# expected broker log
New client connected ... as d:quickstart:mqtt-client:010203060708

Stage 3: Backend MQTT Subscriber (Normalization)

Node backend subscribes to Contiki topic, parses payload, derives normalized fields (temperature, humidity, device_id) and prepares one unified JSON. 

const MQTT_BROKER = process.env.MQTT_BROKER || "mqtt://mosquitto:1883";
mqttClient.subscribe("iot-2/evt/+/fmt/json");

// on message -> normalize
const payload = normalizePayload(topic, message.toString());

Stage 4: Backend WebSocket Broadcast (Bridge)

Backend acts as bridge: every normalized MQTT event is pushed to all active browser clients over persistent WebSocket connections. 

const { WebSocketServer } = require("ws");
const wss = new WebSocketServer({ port: 3000 });

wss.clients.forEach((client) => {
  if (client.readyState === 1) {
    client.send(JSON.stringify(payload));
  }
});

Stage 5: Frontend WebSocket Client (Render)

Dashboard opens WebSocket using current host, receives backend payload, updates live table and protocol logs instantly without polling. 

function getWebSocketUrl() {
  var scheme = window.location.protocol === 'https:' ? 'wss' : 'ws';
  return scheme + '://' + window.location.hostname + ':3000';
}

ws = new WebSocket(getWebSocketUrl());
ws.onmessage = function(event) {
  var data = JSON.parse(event.data);
  updateTable(data);
  addLogEntry('[MQTT -> WS] topic=' + (data.topic || 'n/a'));
};
